# docs

## Purpose
Documentation and API reference for reviewers and evaluators.

## What goes here
- `api.md` — list of all endpoints with example request/response bodies
- `postman_collection.json` — importable Postman collection for testing all endpoints
- `architecture.md` (optional) — brief description of design decisions

## What needs to be done
- Document every endpoint: method, path, request body, response shape, status codes
- Export and include a Postman collection so the reviewer can test with one click
- Add any notes about design decisions (e.g. why standings are calculated dynamically)
